Experiment 003: CORINE Spatial Confidence Re-Evaluation

Deep implementation and result interpretation for article_text_classification_spatial_confidence_v1 — frozen predictions from article_text_classification_e2e_with_shuffled_control_v1, re-scored on CORINE spatial-confidence subsets.

1) What this experiment is and what changed

No LLM re-run

This experiment does not regenerate model outputs. It is a reevaluation pass that keeps all predictions from the parent experiment fixed, computes CORINE spatial-confidence diagnostics for each pageid, and recomputes metrics only on filtered subsets. The goal is to test whether performance gaps between aligned and shuffled text sources grow when only geospatially reliable points are kept.

4) How decisions are implemented in code

Spatial-confidence production

  • Predictions are loaded from the parent directory via JSON prediction files and deduplicated to a single prediction frame for all text sources.
  • Parent CORINE targets are validated as globally consistent across runs; any target mismatch aborts the experiment.
  • Wikipedia points are projected to EPSG:2154 for buffer geometry operations.
  • Spatial confidence is computed as buffer label shares and per-radius dominant share, point-label share, dominant match, entropy, and coverage.
  • Full CORINE is used (including artificial classes), documented by used_full_corine: true.
  • Parquet export is attempted; in this run parquet failed due missing optional dependency and CSV is the authoritative output.

Re-evaluation logic

  • SUBSET_DEFINITIONS includes 6 filters:
    • all available spatial confidence
    • point_label_share_250m >= 0.8
    • point_label_share_500m >= 0.8
    • point_label_share_250m >= 0.9
    • dominant_matches_point_label_250m
    • dominant_matches_point_label_500m
  • Aligned vs shuffled comparison uses paired text-source pairs from TEXT_PAIRS.
  • For CORINE: primary score = balanced_accuracy (macro recall).
  • For OSM: primary score = exact_match_accuracy (multilabel).
  • Shuffled rows are used for delta; no additional model run is performed.

9) Core insight from this experiment

Experiment 3 transforms the interpretation of model quality from "text has little signal" to "text has conditional signal":

  • Spatial-noise filtering matters: only ~52% of pages have point_label_share_250m ≥ 0.8.
  • Precision gets cleaner on higher-confidence subsets: balanced and macro scores improve and deltas vs shuffled widen.
  • Class balance shifts: filtering heavily increases forests, so raw accuracy is not directly comparable without baseline normalization.
  • Implementation robustness: no missing spatial joins and strict consistency checks reduce silent label drift.

In short, the CORINE point coordinates are noisy supervision for many articles. Filtering by spatial confidence rescues this by selecting a better subset rather than claiming global performance.

11) Suggested next experiment in this branch

Given these results, the strongest next step is controlled evidence synthesis for CORINE-relevant classes instead of more subset pruning:

  • Keep dominant_match + point_label_share_250m≥0.8 as a hard reliability gate.
  • Introduce a compact evidence-card text source and compare it against raw content with the same spatial gates.
  • Track class-specific recall for low-performing classes 23/24 before any global optimization.
  • Evaluate whether adding point-label entropy as a weighting term gives better rejection behavior than hard filtering.
