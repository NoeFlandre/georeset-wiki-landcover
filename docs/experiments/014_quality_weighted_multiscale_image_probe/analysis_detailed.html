The Bug We Had To Fix

The first MVP result was invalid because the fetcher called rasterio.dataset.index() with WGS84 longitude/latitude. Sentinel assets use projected raster CRSs, so the coordinates were wrong. With boundless=True, rasterio returned black fill pixels instead of crashing.

Policy Comparison: The Nuanced Story

The clear result is not that soft weighting always beats unweighted training. The clear result is that broad coverage beats aggressive hard filtering.

What Soft Weighting Means

Hard filtering makes a binary decision: keep or drop. Soft weighting keeps broad coverage but changes the contribution of each image during training.

Final Takeaway

Experiment 014 is now valid after the CRS fix. It strengthens the image-learning path: trained frozen-CLIP probes beat zero-shot CLIP strongly. But it also narrows the claim. Soft weighting is promising and helpful at 320 m, yet at 2240 m broad all-unweighted training performs essentially as well. The robust lesson is to keep broad class coverage and avoid aggressive hard filters.
