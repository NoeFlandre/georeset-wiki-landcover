3) Processing flow

flowchart TD A["land-use evidence metadata"] --> B["no-place evidence highlight block"] B --> C["prepend to raw article"] C --> D["content_with_evidence_highlights"] D --> E["Qwen + Gemma CORINE/OSM rerun"] D --> F["shuffled highlighted-content control"] E --> G["compare with raw content, summaries, land-use summary"] F --> G

The highlight block is deterministic. The classifier is rerun on the resulting text sources.

4) What a highlighted-content input looks like

Experiment 010 introduces a highlight block. Unlike the evidence card, it is deliberately minimal: relevance, uncertainty, evidence types, and extracted no-place evidence sentences. It does not include labels, coordinates, spatial purity, predicted labels, or quality scores.

5) What the highlights exclude

IncludedExcludedReason
Land-cover relevanceCORINE / OSM target labelsAvoid direct label leakage.
UncertaintyPredicted labelsKeep the source deterministic and independent of classifier outputs.
Evidence typesCoordinates and place-label sharesAvoid turning spatial metadata into a shortcut.
No-place evidence sentencesQuality scores and recommended-use bucketsTest local text evidence, not the downstream selection score.

12) Implementation notes

  • Highlight builder: evidence_highlights.py.
  • Comparison evaluator: evaluate_evidence_highlights_experiment.py.
  • Text source key: content_with_evidence_highlights; shuffled control: content_with_evidence_highlights_shuffled.
  • The highlight block uses no-place evidence sentences, but the full article follows it unchanged.
  • The classification protocol stays fixed: seed 42, temperature 0.0, same labels, same prompts, same shuffled-control logic.

13) Interpretation and next step

  • Highlighting is a useful diagnostic: aligned-vs-shuffled deltas are strong.
  • It is not a better direct input: raw content still wins most balanced CORINE and OSM comparisons.
  • It improves over short summaries for CORINE: especially compared with landuse_evidence_summary.
  • The next text-only test should be retrieval: classify from the best evidence-bearing raw windows plus context, not from another prepended metadata block.

Recommended next step: move from compact/prepended metadata toward retrieved evidence windows, or use quality and spatial filters for downstream weak-supervision pair selection.
