Experiment 011: Retrieved Evidence Windows

Detailed implementation and result audit for raw sentence windows around matched land-cover evidence.

Qwen + Gemma rerun

Experiment 011 tests whether keeping only raw article sentences around evidence matches preserves most of the full-article signal. It is the strongest compact raw-text representation so far for CORINE, but it still does not replace full raw content.

4) What a retrieved evidence window looks like

Experiment 011 introduces retrieved windows: original article sentences around evidence matches, not a summary. This keeps raw wording and nearby context while shrinking the prompt.

11) Interpretation

  • Retrieval is meaningful: retrieved windows beat random windows and shuffled windows on CORINE.
  • Context matters: sentence-only retrieval is usually weaker than retrieved windows with neighboring sentences.
  • Place names are not the whole story: no-place retrieved windows remain competitive and sometimes improve balanced accuracy.
  • Raw content still wins: full articles contain useful indirect cues outside matched evidence sentences.

The experiment supports using retrieved windows for compact diagnostics or prompt-cost reduction, not as the primary weak-label text source.

12) Implementation notes

  • Window builder: retrieved_evidence_windows.py.
  • Build CLI: build_retrieved_evidence_windows.py.
  • Evaluator: evaluate_retrieved_evidence_windows_experiment.py.
  • Exact sentence matching is attempted first; typed keyword fallback is used when exact evidence sentence matching fails.
  • No classifier labels, predictions, coordinates, spatial scores, or quality scores are injected into classifier text.

13) Next step

The text-source conclusion is now stable across summaries, cards, highlights, and retrieved windows: deterministic evidence artifacts are valuable for diagnostics and cost reduction, while raw content remains strongest for classification.

Recommended next step: freeze candidate weak-supervision article-location pairs using quality, relevance, spatial confidence, and model agreement, then evaluate downstream Sentinel patch training or held-out evaluation.
