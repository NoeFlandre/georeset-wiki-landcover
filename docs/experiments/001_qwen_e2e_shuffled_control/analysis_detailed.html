What the experiment shows: aligned text materially improves performance over shuffled control for both tasks, meaning the signal is driven by article content rather than random priors only.

However, task interpretation differs:

  • CORINE: aligned runs beat a balanced macro-recall baseline even when raw accuracy stays below raw-majority accuracy.
  • OSM: aligned runs beat shuffled controls but remain below strict exact-set majority baseline.
The strongest aligned source in both tasks is raw content, and the most consistent control signal pattern is that shuffled runs collapse toward near-random behavior.

Example pageid 10016815. A shuffled source is not a new wording style: the deterministic control assigns one article's text source to another eligible pageid, preserving the text distribution while breaking article-target alignment.

3.3 Ground truth setup: load_task_setup dispatches two pipelines:
  • corine_level2: build_corine_ground_truth
  • osm: build_osm_ground_truth
3.4 LLM prompting and parsing: LLMClassifier builds schema-constrained JSON prompts, calls complete_json with SINGLE_SCHEMA or MULTI_SCHEMA, then parses via prediction_parser. Unknown labels in OSM JSON can be filtered without failing if all extracted labels are still valid.
3.5 Reproducible shuffled control: apply_shuffled_text_control does a deterministic permutation on eligible IDs using random.Random(seed), tries 100 swaps to avoid self-maps, and then applies a full rotation fallback.
3.6 Resume-safe writes: existing prediction files are read and fingerprint-skipped unless --retry-failed is set. Metrics use only parse_status == ok predictions as evaluated rows.

3.7 CORINE ground-truth exact logic

  • Lat/Lon to WGS84 points via ArticleMeta.
  • Exclude artificial classes with code prefix 1.
  • Convert code_18 to first two digits as level-2 label.
  • Intersect each point with polygons; keep only rows with exactly one unique label for that page.
  • Examples with multi-label or no-label are excluded from target set.

3.8 OSM ground-truth exact logic

  • Derive labels from landuse and natural tags.
  • Keep only polygons with at least one allowed label.
  • Explode multi-label polygons and intersect points.
  • Collect sorted unique labels per page as true target list.

4) Metric definition and what each number means

Single-label CORINE

  • Primary metric is macro_recall computed equally across labels.
  • Primary baseline is 1 / n_unique_labels for CORINE macro recall (0.1111 with 9 labels).
  • accuracy is raw exact-label hit rate.
  • macro_precision / macro_recall / macro_f1 are macro-averages.

Multi-label OSM

  • Primary metric is exact_match_accuracy (full set equality).
  • Macro and micro precision/recall/F1 are also emitted.
  • Per-label values are computed by one-vs-rest binary overlap with support equal to true occurrence count.
  • Majority baseline = most frequent true label-set for exact match, and 1/|labels| for CORINE macro baseline.

Implementation in src/georeset/classification/metrics.py.

Important safety / robustness details

  • coverage = number parsed ok / n_eligible.
  • Evaluation labels = union of true and predicted labels (for macro class list).
  • If parser cannot extract allowed labels, parse status is error and row exits coverage.
  • The pipeline writes each prediction update during iteration, which supports interruption resilience.

Decision artifact

  • classification_policy_version is pinned to 4 in runner fingerprint.
  • Fingerprint includes task, text source, model, seed, temperature, and allowed labels, then SHA-256 hashes into metadata.
  • Summary deltas are computed by pairing aligned/shuffled runs through stable shuffle maps in shuffled_text_source_pairs.

9) Interpretation notes and design decisions

  1. Signal is real, not only priors. For every aligned source and both tasks, aligned score beats shuffled counterpart by large margins (especially CORINE). The control is therefore functioning as an ablation.
  2. CORINE requires imbalance-aware reading. Raw accuracy is below majority accuracy (0.379), but macro-recall is above balanced baseline because minority classes improve, showing better class balance capture.
  3. OSM remains difficult under strict exact-match. Exact-match requires the whole label set to match; this is harsh for open, potentially incomplete label descriptions. Improvements over shuffled indicate signal, but still below majority baseline indicates remaining difficulty in precise set reconstruction.
  4. Why content wins. Removing summaries improves CORINE significantly in this batch, implying generic summarization removes useful land-cover cues.
  5. Class behavior is uneven. Semantically direct classes (e.g., CORINE 22/31/51; OSM water) are recovered better than diffuse classes (pastures, scrub, meadow, orchard).
A major methodological constraint is that model seed and temperature values are effectively static in this experiment path because ModelSettings.from_env() only reads path/repo_id from environment. If you need full parameter search, this is a target for an explicit rerun wrapper.

10) Reconstructable pipeline checklist

If reproducing locally

1. Ensure model files and inputs under data/wiki, data/corine, data/osm.
2. Run command: uv run georeset-classify-articles \
   --task corine_level2 --text-source content --temperature 0.0 \
   --output-dir data/experiments/001_qwen_e2e_shuffled_control/article_text_classification_e2e_with_shuffled_control_v1
3. Repeat all needed combinations: six aligned + six shuffled.
4. Run summary + deltas:
   uv run georeset-summarize-classification-experiment \
     data/experiments/001_qwen_e2e_shuffled_control/article_text_classification_e2e_with_shuffled_control_v1

For exact command signatures and defaults, see runner/CLI modules linked above; for cluster runs, use submit_classification.sh and run_classification_job.sh.
