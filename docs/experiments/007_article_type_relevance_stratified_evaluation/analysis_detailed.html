Experiment 007: Article-Type + Relevance Stratification

Detailed implementation and result audit for using French Wikipedia category metadata as an article-type diagnostic over frozen Qwen and Gemma predictions.

1) Executive summary

No LLM rerun

Experiment 007 asks whether French Wikipedia category metadata helps explain where GeoReset article-text classification works. It reuses frozen predictions, land-use evidence metadata, and spatial confidence, then adds a deterministic category-derived article-type proxy.

Article ids covered

Primary conclusion

Main result: article type is useful as an explanatory diagnostic, but it is not a better filter than land-cover relevance. Category-derived type is incomplete and noisy; evidence relevance remains the stronger quality signal.

11) Implementation notes

  • Evaluator: evaluate_article_type_relevance_stratified.py.
  • Article-type classifier: article_type_classifier.py.
  • Article-type metadata loader: article_type_metadata_loading.py.
  • The assignment uses ordered preference: water, natural landscape, agriculture/vineyard, built/cultural, transport, settlement/admin, person/event, then other/unclear.
  • Category strings are normalized for case, accents, underscores, and French category prefixes before rule matching.

12) Interpretation and next step

  • Article type explains context: landscape, water, and agriculture/vineyard pages are often relevant.
  • Article type is not enough: many pages lack category metadata, and some type buckets are tiny or class-dominated.
  • Relevance remains the main filter: the evidence extractor still finds useful signal inside other_or_unclear.
  • Spatial confidence still matters: the best diagnostic rows combine evidence relevance with reliable point labels.

Recommended next step: use raw content or richer evidence-card representations, then weight or stratify examples by land-cover relevance and spatial confidence rather than relying on category type alone.
