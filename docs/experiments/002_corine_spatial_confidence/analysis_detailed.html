Deep implementation + results explanation for corine_spatial_confidence_v1 and its dependent reevaluation experiment article_text_classification_spatial_confidence_v1.

1) What this experiment is testing

Goal: quantify how spatially reliable each Wikipedia point label is before trusting a CORINE single-label prediction task, then test whether model performance improves on “high-confidence” spatial subsets while keeping predictions frozen.

  1. Spatial confidence generation is orchestrated by src/georeset/cli/data/compute_corine_spatial_confidence.py. Defaults pull:
    • parent frozen predictions from article_text_classification_e2e_with_shuffled_control_v1,
    • wiki page list from data/wiki/wiki_articles.json,
    • CORINE geometry from data/corine/alsace_corine_land_use_2018/occupation_sol_2018.shp.
    The script then loads all pageids from parent prediction files and ensures corresponding point labels and confidence rows align.
  2. Point labels are derived in derive_point_labels() using a spatial join (gpd.sjoin(..., predicate="intersects")) between article point geometries and CORINE polygons.
    • Points without a valid coordinate are dropped.
    • Points mapped to multiple labels are dropped (strict one-label at the point).
    • Parent CORINE targets are reloaded from corine_level2_*_predictions.json and compared to those derived point labels via strict equality.
  3. For each radius in [100, 250, 500, 1000], compute_buffer_label_shares() computes polygon intersections and area fractions in corine_confidence.py. Candidate polygons are accelerated via spatial index on the CORINE GeoDataFrame.
  4. The computed dict is then converted to per-row metric outputs in _radius_metrics():
    • dominant_label, dominant_share
    • point_label_share (primary purity signal)
    • dominant_matches_point_label (dominant class equals point class?)
    • num_labels, entropy, normalized_entropy, coverage_share
    • label_shares_*m serialized as JSON.
  5. Outputs are written as:
    • spatial_confidence.csv (always),
    • manifest.json (run parameters + coverage flags),
    • summary.md (human summary),
    • spatial_confidence.parquet if engine available; currently fallback to CSV only because optional parquet deps are absent.

Primary metrics
OSM

  • dominant_matches_point_label: boolean indicating whether the largest-area class in buffer equals the center point label.
  • point_label_share: proportion of buffer area occupied by the exact point label (core confidence signal).
  • entropy: Shannon entropy over class shares in buffer.
  • normalized_entropy: entropy normalized by number of non-zero labels, range 0-1.
  • coverage_share: sum of buffered intersection shares (can be <1 when buffer is partly outside indexed geometry).

9) Critical interpretation and decisions

  • Decision Strictly keep points whose derived point label matches parent CORINE target. That prevented silent geometry mismatches from propagating downstream.
  • Decision Run in pure post-hoc mode for Experiment 3 (no LLM rerun). This isolates filtering/evaluation effects from generation variance.
  • Decision Use full CORINE in confidence computation and track artificial share separately, instead of masking at source. This preserves observability around urban/built-up contamination.
  • Decision Use fixed subset thresholds (0.8 / 0.9 on point-label share, dominant-match masks). They are pragmatic gates for “high-confidence” spatial quality and were used consistently across text sources.
  • Decision Evaluate OSM and CORINE together for continuity with the prior shuffle-controlled design, even though OSM is stricter exact-match and lower absolute numbers.

Bottom line: Spatial confidence is operationally useful. High-confidence subsets not only increase the apparent reliability (higher point-label purity) but also improve CORINE model quality in controlled aligned-vs-shuffled comparisons, especially in high-purity 250m/500m groups. The same pattern in OSM is directionally positive but small in absolute terms due task hardness.
